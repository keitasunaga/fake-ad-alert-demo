# デモ手順書 - FakeAdAlertDemo

営業デモ用のステップバイステップガイドです。

## 1. デモ準備

### 1.1 Chrome拡張のインストール

1. ターミナルでビルドを実行:
   ```bash
   cd fake-ad-alert-demo
   pnpm install
   pnpm build
   ```

2. Chrome で `chrome://extensions/` を開く

3. 右上の「デベロッパーモード」をONにする

4. 「パッケージ化されていない拡張機能を読み込む」をクリック

5. `dist/` フォルダを選択

6. ツールバーにアイコンが表示されることを確認

### 1.2 事前確認

- Chrome 114 以上であること（Side Panel API対応）
- Instagram / TikTok にログイン済みであること
- 以下のアカウントのページを事前にブックマークしておく:
  - **認証済み**: `@toyota`, `@sony`, `@uniqlo`
  - **フェイク**: ブラックリストに該当するアカウント

## 2. サイドパネルの基本操作

### 2.1 サイドパネルを開く

1. **ツールバーのアイコンをクリック** → ブラウザ右側にサイドパネルが開く
2. DeepL翻訳拡張のようにブラウザ右側に固定表示される
3. ページ遷移しても、タブを切り替えてもサイドパネルは**開いたまま**

### 2.2 サイドパネルの3つの状態

| 状態 | 表示内容 |
|------|---------|
| 認証済み広告検出 | 緑のヘッダー + 4つの展開可能カード |
| フェイク広告検出 | 赤のヘッダー + 広告主情報 |
| 未検出 | 案内メッセージ |

### 2.3 リアルタイム更新

- SNSページで広告が検出されると、サイドパネルの表示が**自動的に更新**される
- ポップアップのように「開き直す」操作は不要
- 別の広告主のページに移動すると、サイドパネルの情報が即座に切り替わる

## 3. Instagram でのデモ

### 3.1 認証済み広告の確認

1. Instagram を開く (`https://www.instagram.com/`)
2. サイドパネルを開いた状態にしておく（アイコンクリック）
3. トヨタ (`@toyota`) のプロフィールページに移動
4. 緑の「✅ VeriCerts認証済み」バッジが表示されることを確認
5. **サイドパネルが自動更新**され、VC検証情報が表示されることを確認
6. 4つの展開可能カードを順に開いて説明:
   - **広告情報**: 広告主名、DID、カテゴリ
   - **検証ステータス**: 5項目すべて✓（署名、有効期限、失効状態、トラストレジストリ、ブロックチェーン）
   - **信頼チェーン**: 消費者庁 → トラスト広告社 → トヨタ自動車
   - **ブロックチェーン証明**: Sepolia ネットワーク上のトランザクション

### 3.2 フェイク広告の確認

1. ブラックリストに該当するアカウントのページに移動
2. 赤い「⚠️ 未認証広告の可能性」警告が表示されることを確認
3. **サイドパネルが自動更新**され、フェイク警告が表示されることを確認

## 4. TikTok でのデモ

### 4.1 認証済みアカウント

1. TikTok を開く (`https://www.tiktok.com/`)
2. ソニー (`@sony`) のプロフィールページに移動
3. 緑の認証バッジが表示されることを確認
4. グリッド内の各投稿にも緑のバッジが表示される
5. 個別の動画ページでもユーザー名横にバッジが表示される
6. **サイドパネルが自動更新**され、VC検証情報を確認

### 4.2 フェイクアカウント

1. ブラックリストに該当するアカウントのページに移動
2. 赤い警告が表示されることを確認
3. **サイドパネルが自動更新**され、フェイク警告を確認

## 5. デモでのポイント

- 各カードは**クリックで展開/折りたたみ**可能
- DID（`did:web:toyota.co.jp`）がブロックチェーン上のアイデンティティであることを説明
- 信頼チェーンが「消費者庁 → 認定機関 → 広告主」の階層であることを強調
- share-verifierアプリと**同じ検証項目**を表示していることを説明
- サイドパネルは**ページ遷移しても閉じない**ため、複数アカウントの確認がスムーズ
- 広告検出時に**リアルタイムで自動更新**されるため、操作の手間がない

## 6. share-verifier との連携デモ

1. FakeAdAlertDemo で広告を検出した後
2. share-verifier アプリを開く
3. 同じ広告主の証明書情報が同じ構造で表示されることを確認
4. 「ブラウザ拡張とモバイルアプリで統一された検証体験」をアピール

## 7. トラブルシューティング

### 拡張機能が動作しない
- `chrome://extensions/` で拡張機能が有効か確認
- 拡張機能の更新ボタンをクリック
- ページをリロード

### バッジが表示されない
- SNSページのDOM構造が変更されている可能性
- コンソール（F12）でエラーを確認
- 拡張機能を再読み込み

### サイドパネルに「未検出」と表示される
- まずSNSページで広告を表示させる（サイドパネルは自動更新される）
- サイドパネルは直近の検出情報のみ表示

### サイドパネルが開かない
- Chrome 114 以上であることを確認
- ツールバーにアイコンがあるか確認
- なければ拡張機能のパズルアイコンからピン留め
